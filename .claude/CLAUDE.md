# Mnemosyne — Chrome Extension

> Бывш. **ClaudeVoiceMaster (CVM)**. Проект переименован в **Mnemosyne**
> (греческая богиня памяти и слова) на стадии 0.0.2 и развивается в универсальный
> инструмент перевода и нейронной озвучки видео, а не только клиент под YouTube.

## Концепция проекта

Mnemosyne — Chrome-расширение для просмотра видео на любом языке.
Расширение автоматически переводит субтитры и озвучивает перевод нейронным
голосом синхронно с видео, создавая полноценный опыт просмотра на родном языке.

**Текущий статус: разработка alpha 0.0.2.** База из alpha 0.0.1 (перевод +
нейронная озвучка Edge + караоке-субтитры) работает. Планы версии — в
`.claude/TASKS(v_0.0.2_alpha).md`; журнал alpha 0.0.1 — в
`.claude/TASKS(v_0.0.1_alpha).md`.

Ключевые принципы:
- Перевод делается заранее (до воспроизведения), не в реальном времени
- Субтитры YouTube — источник текста (ASR не требуется)
- TTS движок абстрагирован для лёгкой замены без переписывания логики
- Developer tools — отдельная страница расширения, не часть popup

## Технический стек

- **Framework**: WXT (Web Extension Tools) — Manifest V3, TypeScript, HMR
- **Язык**: TypeScript (строгий режим)
- **API перевода**: Claude через aiprimetech.io
- **TTS**: Edge TTS (нейронный, через Cloudflare Worker) + Web Speech (запасной);
  далее → CosyVoice 3 / Qwen3-TTS / Yandex SpeechKit
- **Сборщик**: встроен в WXT (Vite)
- **Тестирование**: запуск через `npm run dev` (WXT открывает Chrome автоматически)

## API

```
Base URL:   https://aiprimetech.io
Endpoint:   https://aiprimetech.io/v1/messages
Model:      claude-sonnet-4-6
Совместим с официальным Anthropic API (drop-in replacement)
```

API ключ хранится только в `chrome.storage.local`. Никогда не хардкодить.

│   ├── storage.ts                  ← типизированные storage.defineItem

│   ├── tts.ts                      ← абстракция TTS-движка: WebSpeech / Edge / Composite

> **Примечание по именам:** папка `inspector/` (а не `devtools/`) — имя `devtools`
> в WXT зарезервировано под настоящую панель Chrome DevTools. Контент-скрипт моста
> именуется `content-bridge.content.ts` — WXT распознаёт контент-скрипты только
> по шаблону `*.content.ts`.

## Стадии разработки

- Описаны в актуальном TASKS в `.claude/` (текущая версия — `TASKS(v_0.0.2_alpha).md`)
- Там же ведётся запись ключевых моментов на всех стадиях разработки (в разделе каждого этапа)
- Файлы `.claude/CLAUDE.md` и `TASKS(...).md` редактируются только по запросу самого пользователя
- По просьбе пользователя или при необходимости дополнять стадии разработки в формате "Стадия 2.x"

## Инструкции для Claude Code

### Как работать с пользователем

1. **Обсуждай перед кодом.** Перед любым изменением кратко опиши что именно
   собираешься изменить и почему. Жди подтверждения на сложных решениях.

2. **Одна задача за раз.** Не трогай файлы не относящиеся к текущей задаче.
   Если видишь попутный баг — упомяни его, но не фикси без команды.

3. **Сообщай о результате.** После выполнения задачи коротко опиши что сделано
   и как проверить результат (какой URL открыть, что нажать).

4. **Стадийная разработка.** Текущая стадия — приоритет. Не забегай вперёд
   и не реализуй функционал следующей стадии без команды.

5. **TypeScript строго.** Никаких `any`. Все типы явные.

6. **Без магических констант.** Все числа и строки — именованные константы
   в верхней части файла или в отдельном `constants.ts`.
   
7. **Запрет на git** - Работать с git, создавать комиты, пушить можно только
   по просьбе пользователя. 

### Порядок проверки задачи

1. Прочитай актуальный TASKS в `.claude/` — найди текущую задачу
2. Объясни план изменений
3. Выполни
4. Опиши как проверить
