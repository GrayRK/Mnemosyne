# ClaudeVoiceMaster (CVM → Mnemosyne) — Tasks V_0.0.2_alpha

## Замысел версии

> Версия 0.0.2 — это переосмысление, а не просто доработка. После изучения готовых
> аналогов (в первую очередь **VOT** на инфраструктуре Яндекса — см.
> [Analiz_VOT.md](./Analiz_VOT.md)) стало ясно: базовый сценарий «перевести и озвучить
> YouTube» уже решён другими, и местами быстрее. Поэтому цель версии — **найти, ради чего
> проект существует**, и отстроиться:
>
> - **Ренейм в «Mnemosyne»** (греческая богиня памяти и слова) — проект становится мощным
>   многофункциональным инструментом озвучки, а не клоном VOT.
> - **Сильные стороны для отстройки**: качество перевода (Claude и др. LLM), гибкость
>   голосов, бесплатный дефолт без API, локальные LLM в браузере, охват платформ за
>   пределами YouTube.
> - **Архитектурные рычаги скорости из VOT** (см. Analiz_VOT): общий серверный кэш, батч,
>   async pre-render — берём из архитектуры, а не из паразитирования на чужом API.
>
> Последовательность выполнения **не привязана к номеру стадии** — этапы берутся в работу
> по приоритету и зависимостям (см. «Приоритеты выполнения»).

## Стадия 2 — Данные, инфраструктура, охват

**Статус**: ⬜ Не начата
**Цель**: расширить источники данных из видео, инфраструктуру обхода блокировок и общий кэш,
охватить новые плееры, платформы и браузеры.

### Задача

**2.1 — Источники данных видео.** Что вообще можем получить и зачем:
- **Transcript (расшифровка)**: можно ли дёргать, нужно ли, как поведут себя таймкоды.
- **Аудиодорожка**: возможность извлечения (минимум — под имитацию/клонирование голоса в
  мощных TTS, Стадия 4).
- **Диаризация** (кто какую фразу произносит): вероятно недоступно, но необходимо для
  многоголосой озвучки — исследовать.

**2.2 — Обход блокировок (прокси).**
- Учесть два типа блокировок: (1) **от сервиса** — обход прокси/подменой IP/сетевыми
  средствами; (2) **DPI в РФ** — проще, за основу можно взять подход проекта Zapret (обман DPI).
- Userfriendly встроенный обход «из коробки» + возможность подключить **личный прокси**.
- Сейчас Edge TTS идёт через свой Cloudflare Worker — рассмотреть как часть этой инфраструктуры,
  поискать альтернативы.

**2.3 — Общее хранилище переводов.** (Ключевой рычаг, см. [Analiz_VOT.md](./Analiz_VOT.md), рычаг №1)
- Хранить готовые переводы/озвучку **не только локально**, чтобы пользователи переиспользовали
  чужие результаты (по аналогии с серверным кэшем Яндекса/VOT).
- Ключ записи: `videoId + targetLang + voice` (расширить Cloudflare Worker под KV/R2).
- Модель: пользователь авторизуется в сервисе-посреднике (подобрать бесплатный) → даёт доступ
  на запись → выполненный перевод пишется в общее хранилище **и** локально → другой пользователь
  того же видео получает готовое мгновенно.
- Рассмотреть варианты: **единое хранилище** vs **множество пользовательских (P2P)** через
  сервис-посредник.

**2.4 — Метрики и стоимость.** (Закрытие долга 0.0.1; UI-часть — в 1.4)
- Довести метрики до нормального, законченного вида.
- Отказаться от концепции калькулятора как «прикинуть примерно» и «считать по данным
  пользователя» — показывать **реальную стоимость** перевода.
- Объединить метрики с расчётом стоимости → единые метрики с ценой; проверить, можно ли
  запрашивать нужные данные напрямую у API (`usage`).

**2.5 — Охват плееров и платформ.**
- Уйти от жёсткой привязки к плееру YouTube: определять другие популярные проигрыватели и
  YouTube-плеер, встроенный на сторонних сайтах.
- Цели: Netflix, Amazon Prime, Disney+, и др. (по возможности).

**2.6 — Стримы (live).**
- Реал-тайм сценарий (отдельный вызов относительно «перевод заранее»): Twitch, YouTube Live и др.

## Стадия 3 — Перевод

**Статус**: ⬜ Не начата
**Цель**: множество способов перевода — от бесплатного дефолта без ключа до локальных LLM
в браузере и системе.

### Задача

**3.1 — Выбор способа перевода (архитектура).** Дать пользователю выбор источника перевода:
- **Google Translate** — по умолчанию, встроенный (автоперевод вместо флага). См. 3.2.
- **API** разных форм. См. 3.3.
- **Локальные LLM встроенные** — ультралёгкие модели внутри браузера. См. 3.6.
- **Локальные LLM системные** — лёгкие модели в системе пользователя. См. 3.7.

**3.2 — Google Translate (бесплатный дефолт).**
- Протестировать, нормально ли работает с расшифровкой из 2.1.
- Сделать автопереводом по умолчанию вместо текущего флага «использовать автоперевод YouTube».

**3.3 — API: мультипровайдер.**
- Возможность использовать **множество API** одновременно.
- Встроенные бесплатные провайдеры: исследовать, на чём можно ехать бесплатно (например
  **OpenRouter** с бесплатными моделями по умолчанию + подключение своего аккаунта); другие
  агрегаторы помимо OpenRouter.
- Юзерфрендли авторизация на сервисах; редактируемый системный промпт перевода.
- Работа с официальными API (Claude, OpenAI и др.).

**3.4 — API: ускорение и качество промпта.**
- Дальнейшее ускорение запросов: потоковая загрузка одним батчем / уменьшение числа батчей —
  продолжить оптимизацию.
- Улучшение системного промпта перевода.
- **Контекст из названия и описания видео** в промпте (они уже есть на странице) — для более
  точного перевода терминов, тона, имён.
- 🐞 **Баг (отложен):** кусок озвучки на языке оригинала. На переводе через Claude API
  небольшой фрагмент (~середина) озвучен на оригинале. Вероятная причина: толерантная сборка
  `parseNumbered` (`background.ts`) подставляет оригинал строки при потере её номера — такие
  строки уходят в озвучку как есть. Проверить логом «строк осталось оригиналом», при
  подтверждении — точечный ре-перевод выпавших номеров.

**3.5 — Подготовка текста к озвучке.** (Стык перевода и TTS — пересекается со Стадией 4)
- Собирать полноценные фразы; чистить/нормализовать артефакты, ломающие озвучку: `[СМЕХ]`,
  `[АПЛОДИСМЕНТЫ]`, «э-э», «м-м» и т.п.
- Возможно задать стандарт служебных ремарок и обрабатывать в TTS (например, проигрывать звук
  хлопков на `[АПЛОДИСМЕНТЫ]`).

**3.6 — Локальные LLM встроенные (браузер).** (Главный дифференциатор — «самое вкусное»)
- Модели на **WebGPU**: Transformers.js, WebLLM, Qwen 0.6B — всё, что можно запустить из браузера.
- Дозагрузка модели: к проекту модель не прилагается, но по кнопке «Установить» подтягивается
  нужная (расширение для расширения).

**3.7 — Локальные LLM системные.** (Отложено)
- Пока нет удобного способа запускать процесс вне браузера (в системе), что нужно для более
  мощных LLM. Возможно позже примем ручной запуск и реализуем.
