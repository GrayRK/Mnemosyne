# ClaudeVoiceMaster (CVM → Mnemosyne) — Tasks V_0.0.2_alpha

## Замысел версии

> Версия 0.0.2 — это переосмысление, а не просто доработка. После изучения готовых
> аналогов (в первую очередь **VOT** на инфраструктуре Яндекса — см.
> [Analiz_VOT.md](./Analiz_VOT.md)) стало ясно: базовый сценарий «перевести и озвучить
> YouTube» уже решён другими, и местами быстрее. Поэтому цель версии — **найти, ради чего
> проект существует**, и отстроиться:
>
> - **Ренейм в «Mnemosyne»** (греческая богиня памяти и слова) — проект становится мощным
>   многофункциональным инструментом озвучки, а не клоном VOT.
> - **Сильные стороны для отстройки**: качество перевода (Claude и др. LLM), гибкость
>   голосов, бесплатный дефолт без API, локальные LLM в браузере, охват платформ за
>   пределами YouTube.
> - **Архитектурные рычаги скорости из VOT** (см. Analiz_VOT): общий серверный кэш, батч,
>   async pre-render — берём из архитектуры, а не из паразитирования на чужом API.
>
> Последовательность выполнения **не привязана к номеру стадии** — этапы берутся в работу
> по приоритету и зависимостям (см. «Приоритеты выполнения»).

- **Разметка**: статические строки помечены `data-i18n` (на листовых узлах, чтобы не затирать

## Стадия 5 — Нативный хэлпер (десктоп-компаньон)

**Статус**: ⬜ Не начата
**Цель**: вынести тяжёлую работу из браузера в нативное приложение-компаньон, потому что
ключевые задачи версии **принципиально не решаются в рамках расширения**: добыча медиа
упирается в защиту YouTube (poToken/BotGuard, SABR), а полноценные локальные LLM не влезают
в ограничения WebGPU. Расширение остаётся **тонким клиентом и работает в лайт-режиме без
хэлпера** (текущий функционал — дефолт), а установленный хэлпер разблокирует полную мощь.

> **Сценарий для пользователя**: скачал exe-инсталлятор с релиза → установил (путь, настройки) →
> хэлпер встаёт службой в систему, регистрирует связь с расширением → браузер предлагает
> установить расширение → расширение находит хэлпер и соединяется.
>
> **Это меняет идентичность проекта**: из «Chrome-расширения» Mnemosyne становится
> «десктоп-компаньон + расширение». Для кода расширения правки аддитивные (новый бэкенд за
> существующими абстракциями + graceful degradation); основной объём новой работы — в хэлпере,
> упаковке и релизах. Архитектурный паттерн стандартный: Native Messaging host + локальный сервис.

### Задача

**5.1 — Архитектура и связка «расширение ↔ хэлпер».**
- Реорганизация репозитория в много-компонентный: `extension/` (текущий код WXT), `helper/`
  (новое нативное приложение), `installer/` (упаковка). `worker/` остаётся (см. 5.x / 2.3).
- Транспорт: локальный сервер на `127.0.0.1` (WS/HTTP) для тяжёлого и потокового обмена +
  Native Messaging для безопасного «рукопожатия» / обнаружения хэлпера.
- Клиент хэлпера в расширении; индикатор статуса в popup (подключён / не установлен / версия).
- **Лайт-режим обязателен**: расширение полностью работает без хэлпера; хэлпер лишь разблокирует
  мощные опции в capability-driven UI (механика выбора уже заложена на этапе 1.3).
- Безопасность: слушать только `127.0.0.1`, токен на соединение, проверка ID расширения
  (учесть, что у dev/unpacked-сборки и у сборки из Web Store **разные ID** — оба в allow-list).

**5.2 — Хэлпер: добыча медиа (yt-dlp + ffmpeg).**
- Бандл **yt-dlp** + **ffmpeg**; добыча аудиодорожки/видео оригинала по `videoId`. Это закрывает
  потребность, которую браузер не тянет из-за защиты YouTube (ограничения зафиксированы в 2.1) —
  чужим поддерживаемым инструментом, без собственного арм-рейса.
- Модель «задача → прогресс → результат» (по образцу VOT, см. [Analiz_VOT.md](./Analiz_VOT.md)):
  расширение ставит задачу, хэлпер отдаёт прогресс и готовый артефакт.
- Отдача файла/субтитров расширению для синхронной озвучки и кэширования.

**5.3 — Хэлпер: локальные LLM (перевод / ASR / TTS).**
- Полноразмерные модели вне ограничений WebGPU: ASR (whisper / faster-whisper), перевод,
  нейронная озвучка с клонированием голоса (**CosyVoice**, **Chatterbox** и др.).
- Делает **реализуемыми отложенные этапы 3.7 и 4.5** (системные LLM): хэлпер и есть тот самый
  «способ запускать процесс вне браузера», которого не хватало.
- Менеджмент моделей: дозагрузка по запросу, выбор модели/качества под железо пользователя.

**5.4 — Инсталлятор и служба (Windows, первая платформа).**
- exe-инсталлятор (Inno Setup / NSIS): путь установки, базовые настройки.
- Регистрация хэлпера как **службы Windows** (или автозапуска) — «всегда доступен» для долгих
  задач и локального LLM-сервера.
- Запись native-messaging-манифеста в реестр (`HKCU\...\NativeMessagingHosts\<имя>`) со списком
  разрешённых ID расширения.
- После установки — открытие страницы установки расширения.

**5.5 — Подпись кода и доверие.**
- Подпись бинаря и инсталлятора, чтобы не блокировали SmartScreen / антивирусы.
- Прозрачность: что именно ставится в систему и зачем (служба, манифест, модели).

**5.6 — Авто-обновление хэлпера.**
- Нативное приложение обновляет себя само (не связано запретом MV3 на удалённый код) — канал
  обновлений через релизы GitHub. Самое волатильное (напр. yt-dlp) обновляется отдельно/штатно.

**5.7 — Кроссплатформенность.**
- macOS / Linux: их способы установки службы (launchd / systemd) и регистрации
  native-messaging-манифеста; сборка инсталляторов под каждую ОС в релиз-пайплайне.

**5.8 — Тестирование и отладка.**
- Хэлпер тестируется **в изоляции, без браузера**: локальный сервер + юнит-тесты (yt-dlp, LLM,
  модель задач) — то, чего не хватало в браузерном тангле.
- Dev-связка: хэлпер запускается из исходников рядом с `npm run dev`; dev-ID расширения
  прописан в allow-list хэлпера; при ребуте перезапускать оба компонента.
- Два потока логов: расширение — в DevTools, хэлпер — в своём терминале/файле; запрос трассируется
  extension → helper → обратно.
- Путь «инсталлятор + служба» — отдельная **релизная** проверка (чистая Windows/VM), не часть
  быстрого внутреннего цикла.
