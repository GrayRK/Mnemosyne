# ClaudeVoiceMaster (CVM) — Chrome Extension

## Концепция проекта

CVM — Chrome-расширение для просмотра YouTube видео на любом языке.
Расширение автоматически переводит субтитры и озвучивает перевод в реальном
времени, создавая полноценный опыт просмотра на родном языке пользователя.

Ключевые принципы:
- Перевод делается заранее (до воспроизведения), не в реальном времени
- Субтитры YouTube — источник текста (ASR не требуется)
- TTS движок абстрагирован для лёгкой замены без переписывания логики
- Developer tools — отдельная страница расширения, не часть popup

---

## Технический стек

- **Framework**: WXT (Web Extension Tools) — Manifest V3, TypeScript, HMR
- **Язык**: TypeScript (строгий режим)
- **API перевода**: Claude через aiprimetech.io
- **TTS**: Edge TTS (старт) → CosyVoice 3 / Qwen3-TTS / Yandex SpeechKit
- **Сборщик**: встроен в WXT (Vite)
- **Тестирование**: запуск через `npm run dev` (WXT открывает Chrome автоматически)

## API

```
Base URL:   https://aiprimetech.io
Endpoint:   https://aiprimetech.io/v1/messages
Model:      claude-sonnet-4-6
Совместим с официальным Anthropic API (drop-in replacement)
```

API ключ хранится только в `chrome.storage.local`. Никогда не хардкодить.

## Инструкции для Claude Code

### Как работать с пользователем

1. **Обсуждай перед кодом.** Перед любым изменением кратко опиши что именно
   собираешься изменить и почему. Жди подтверждения на сложных решениях.

2. **Одна задача за раз.** Не трогай файлы не относящиеся к текущей задаче.
   Если видишь попутный баг — упомяни его, но не фикси без команды.

3. **Сообщай о результате.** После выполнения задачи коротко опиши что сделано
   и как проверить результат (какой URL открыть, что нажать).

4. **Стадийная разработка.** Текущая стадия — приоритет. Не забегай вперёд
   и не реализуй функционал следующей стадии без команды.

5. **TypeScript строго.** Никаких `any`. Все типы явные.

6. **Без магических констант.** Все числа и строки — именованные константы
   в верхней части файла или в отдельном `constants.ts`.
   
7. **Запрет на git** - Работать с git, создавать комиты, пушить можно только
   по просьбе пользователя. 

### Порядок проверки задачи

1. Прочитай TASKS.md — найди текущую задачу
2. Объясни план изменений
3. Выполни
4. Опиши как проверить
